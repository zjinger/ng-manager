# Hub-v2 Migration Index (0041-0049)

This index documents the intent and dependency order for the current admin/RBAC/organization/expense foundation migrations.

## Ordered Scope

1. `0041_user_org.sql`  
   Organization base: `departments`, final-form `user_departments` (single primary department per user).

2. `0042_system_rbac.sql`  
   Unified RBAC base: `system_roles`, `system_permissions`, role bindings, initial seeds.

3. `0043_approval.sql`  
   Approval template and stage metadata (fixed-stage template model).

4. `0044_reimbursement.sql`  
   Reimbursement core tables: claims, items, attachments, tasks, logs.

5. `0045_project_rbac.sql`  
   Project-governance related permission seeds into `system_permissions`.

6. `0046_system_settings.sql`  
   Global system settings storage.

7. `0047_system_titles.sql`  
   Global title dictionary (`system_titles`) with active/inactive status.

8. `0048_department_titles.sql`  
   Department-to-title mapping (`department_titles`).

9. `0049_system_permissions_status.sql`  
   Adds permission lifecycle fields (`status`, `is_builtin`) for dynamic permission-item management.

## Notes

- `0047_user_departments_single_primary.sql` was removed before deployment, and subsequent migrations were renumbered to keep a clean contiguous sequence.
- No business semantics were changed in this cleanup; this round is sequence and maintainability cleanup only.
