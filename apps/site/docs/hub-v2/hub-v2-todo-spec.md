# Hub v2 — 个人待办事项功能需求规格

## 一、功能概述

在 Hub v2 内网协作平台中新增「个人待办事项」模块，用于管理用户的日常任务。作为个人效率工具，独立于项目的问题跟踪系统，数据归属个人。

## 二、页面布局

遵循 Hub v2 现有布局结构：

- **左侧导航栏**：在「工作台」分组下新增 `个人待办` 菜单项，图标 `check-circle`，右侧显示未完成任务数量的红色 badge
- **顶部栏**：面包屑显示 `首页 > 个人待办`，右侧保留搜索框、通知、帮助按钮
- **内容区**：统计卡片 → 工具栏 → 任务列表/看板

## 三、功能模块详述

### 3.1 统计概览（页面顶部）

4 张统计卡片横排展示：

| 卡片 | 图标 | 图标色 | 说明 |
|------|------|--------|------|
| 全部任务 | schedule| blue (#3B82F6) | 显示任务总数 |
| 进行中 | sync | purple (#7C3AED) | 状态为「进行中」的数量，下方蓝色进度条 |
| 已完成 | check-circle | green (#10B981) | 状态为「已完成」的数量，下方绿色进度条 |
| 已逾期 | exclamation-triangle | red (#EF4444) | 截止日期早于今天且未完成的数量 |

进度条计算方式：该状态数量 / 总任务数 × 100%

### 3.2 工具栏

从左到右排列：

1. **新建任务按钮**（btn-primary）：打开新建任务弹窗
2. **筛选器**（3 个下拉 select）：
   - 状态筛选：全部状态 / 待办 / 进行中 / 已完成
   - 优先级筛选：全部优先级 / 紧急 / 高 / 中 / 低
   - 标签筛选：全部标签 / 工作 / 个人 / 学习 / 紧急
3. **搜索输入框**：实时搜索任务标题和描述，支持 `⌘K` 快捷聚焦
4. **视图切换**（toggle 按钮组）：列表视图 / 看板视图
5. **清除已完成**（btn-ghost）：批量清除所有已完成任务，带确认弹窗

### 3.3 列表视图（默认视图）

以卡片形式包裹，卡片头部显示「任务列表」标题和当前筛选后的任务数量 badge。

每条任务行结构：

```
[完成圆圈] [优先级色条] [任务内容区]                                    [操作按钮(悬浮显示)]
```

**完成圆圈**：
- 未完成：灰色空心圆，hover 变蓝
- 已完成：绿色实心圆 + 白色勾
- 点击切换完成/恢复

**优先级色条**（宽 4px 圆角条）：
- 紧急 critical → 红色 (#EF4444)
- 高 high → 橙色 (#F59E0B)
- 中 medium → 蓝色 (#3B82F6)
- 低 low → 灰色 (#CBD5E1)

**任务内容区**：
- 第一行：任务标题 + 状态徽标（待办/进行中/已完成/已逾期）
- 第二行（可选）：描述文本，单行截断，灰色
- 第三行：优先级指示器 + 截止日期 + 标签徽标

**状态徽标样式**：
- 待办 → 灰底灰字
- 进行中 → 蓝底蓝字
- 已完成 → 绿底绿字
- 已逾期 → 红底红字（当截止日期 < 今天且未完成时显示）

**截止日期显示规则**：
- 今天 → 「今天」
- 明天 → 「明天」
- 昨天 → 「昨天」
- 7 天内 → 「X天后」/「X天前」
- 超出 7 天 → 「M月D日」
- 逾期日期显示红色

**标签徽标**：
- 工作 → 蓝底
- 个人 → 紫底
- 学习 → 绿底
- 紧急 → 红底

**操作按钮**（hover 任务行时显示）：
- 编辑（fa-pen）→ 打开编辑弹窗
- 删除（fa-trash-alt）→ 打开确认弹窗

**空状态**：无任务时显示图标 + 「暂无任务」文案 + 新建任务按钮

**排序规则**：
1. 先按状态：进行中 → 待办 → 已完成
2. 同状态按优先级：紧急 → 高 → 中 → 低
3. 同优先级按截止日期升序
4. 无截止日期排最后

### 3.4 看板视图

三列看板：待办 | 进行中 | 已完成

每列：
- 列头：状态颜色圆点 + 状态名 + 任务数量 badge
- 卡片内容：标题（最多 2 行截断）+ 描述（最多 2 行截断）+ 底部（标签 + 截止日期）
- 支持拖拽卡片到其他列，自动更新任务状态
- 拖拽时目标列高亮虚线边框

### 3.5 新建/编辑任务弹窗

弹窗尺寸：max-width 580px

表单字段：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 任务标题 | text input | ✅ | 最大 100 字符，Enter 快捷保存 |
| 任务描述 | textarea | ❌ | 最大 500 字符 |
| 优先级 | select | ✅ | 低/中/高/紧急，默认「中」 |
| 截止日期 | date input | ❌ | 日期选择器 |
| 状态 | select | ❌ | 待办/进行中/已完成，默认「待办」 |
| 标签 | chip 多选 | ❌ | 工作/个人/学习/紧急，可多选，点击切换选中态 |

弹窗行为：
- 新建模式：标题「新建任务」，所有字段清空
- 编辑模式：标题「编辑任务」，回填已有数据
- 标题为空时 toast 提示「请输入任务标题」
- Esc 关闭弹窗
- 点击遮罩关闭弹窗
- 保存后自动刷新列表

### 3.6 确认弹窗

用于删除单个任务和批量清除已完成任务。

结构：感叹号图标 + 标题 + 描述文案 + 取消/确认按钮

### 3.7 Toast 通知

右上角弹出，2.5 秒后自动消失。三种类型：
- success：绿色左边框 + 勾号图标
- error：红色边框 + 叉号图标
- info：蓝色边框 + 信息图标

触发场景：
- 创建任务 → success「任务创建成功」
- 编辑保存 → success「任务已更新」
- 删除任务 → info「任务已删除」
- 完成任务 → success「任务已完成 ✅」
- 恢复任务 → success「任务已恢复」
- 拖拽状态变更 → success「任务已移至「XXX」」
- 标题为空 → error「请输入任务标题」

## 四、数据模型

```typescript
interface Todo {
  id: string;           // 唯一 ID 
  title: string;        // 任务标题
  desc?: string;        // 任务描述
  priority: 'low' | 'medium' | 'high' | 'critical';
  status: 'todo' | 'doing' | 'done';
  due?: string | null;  // 截止日期 YYYY-MM-DD
  tags: string[];       // 标签数组 ['work', 'personal', 'study', 'urgent']
  createdAt: string;    // 创建日期 YYYY-MM-DD
}
```

## 五、交互细节

### 5.1 键盘快捷键
- `Enter`：在标题输入框中按 Enter 快捷保存

### 5.2 侧边栏 Badge
- 实时显示未完成任务数量（状态不为 done 的任务数）
- 数量为 0 时隐藏 badge

### 5.3 逾期判定逻辑
- `due < 今天` 且 `status !== 'done'` → 逾期
- 已完成的任务不标记逾期

## 六、样式规范

完全复用 Hub v2 现有设计系统：

- **CSS 变量**：使用 `--primary-*`、`--gray-*`、`--success`、`--warning`、`--danger`、`--info` 等现有变量
- **圆角**：容器 `--border-radius (8px)`，小组件 `--border-radius-sm (6px)`
- **阴影**：`--shadow-sm` / `--shadow` / `--shadow-md` / `--shadow-lg` / `--shadow-xl`
- **字体**：同 Hub v2，`-apple-system, BlinkMacSystemFont, 'PingFang SC'...`
- **过渡**：`--transition: all 0.2s ease`
- **组件复用**：btn、card、filter-select、search-input、status-badge、modal、form-input 等与 Hub v2 保持一致

## 七、技术要求

- 首次加载无数据时预填示范数据（约 9 条，覆盖不同状态/优先级/标签）
- 支持拖拽排序（看板视图）
- 所有文本输入需做 XSS 防护（转义 HTML）
