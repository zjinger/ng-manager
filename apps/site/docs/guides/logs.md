# 日志与排障

日志是 `ng-manager` 里最快速的诊断入口。无论是任务失败、服务未启动，还是脚本异常退出，先看日志。

## 建议的排查顺序

1. 看任务最近一次输出，确认报错发生在哪个阶段。
2. 看退出码，判断是业务失败还是进程被外部中断。
3. 看系统日志，确认是否存在端口、权限或依赖相关问题。

## 常见故障

### 端口被占用

- 表现：开发服务无法启动，日志提示端口冲突。
- 处理：先停止旧进程，再重试任务；必要时调整启动端口。

### Node 版本不匹配

- 表现：旧项目启动时报语法或依赖安装错误。
- 处理：切换到项目兼容的 Node 版本，再重新执行。

### 依赖未安装或锁文件漂移

- 表现：模块缺失、构建失败、脚本无法解析。
- 处理：在项目目录重新安装依赖，并确认锁文件与团队基线一致。

## 什么时候用 CLI 辅助排查

如果 UI 无法正常拉起，先用 CLI 检查服务状态，再决定是重启服务还是直接看本地运行日志。
