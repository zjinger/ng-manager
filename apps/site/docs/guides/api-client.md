# API 调试

API 页面提供一个内置请求工作台，用于管理请求集合、环境变量、发送请求和查看历史。

## 页面结构

- 左侧：请求集合与目录树。
- 右侧上半区：请求编辑器。
- 右侧下半区：响应查看器。
- 顶部操作区：环境变量切换、历史记录入口。

## 主要能力

- 新建请求、集合和文件夹。
- 重命名、移动、删除集合项。
- 编辑 URL、请求头、鉴权、请求体等内容。
- 绑定环境变量作为基础地址或动态参数。
- 查看历史记录并重放请求。

## 适合的使用场景

- 团队希望把常用接口测试沉淀在项目内，而不是散落在个人工具里。
- 本地联调时需要在同一工作台里切换开发服务和接口请求。
- 想要用比 Postman 更贴近项目上下文的轻量调试入口。

## 使用建议

- 先配置环境变量，再维护请求集合，避免 URL 到处硬编码。
- 把常用接口按模块分组，减少集合树膨胀。
- 调试完成后保留有价值的历史请求，方便后续复现问题。
