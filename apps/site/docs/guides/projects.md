# 项目管理

`ng-manager` 的核心对象是“本地项目”。只要项目能在本机运行，它就适合被纳入统一管理。

## 导入方式

- 选择现有工程目录导入。
- 让系统扫描项目根目录，识别脚本、框架与可执行能力。
- 对于多仓库或多应用目录，优先从真正的项目根目录开始导入。

## 导入后会识别什么

- `package.json` 中定义的脚本。
- 常见框架特征，例如 Angular、Vue、Node。
- 与任务运行相关的目录、依赖和启动入口。

## 建议的项目整理方式

- 保持每个项目的根目录清晰，避免把无关目录混在同一层级。
- 先确认本地能独立运行，再交给 `ng-manager` 托管。
- 对历史项目，先对齐 Node 版本，再做导入，减少首次扫描误判。

## 什么时候需要重新扫描

- 你新增或修改了 `package.json` scripts。
- 项目升级了构建工具或框架。
- 团队引入了新的启动命令、端口约定或环境变量。
