# 仪表盘

仪表盘是进入 `ng-manager` 后最适合长期停留的首页。它面向“当前项目”的高频操作，支持按需摆放部件。

## 页面定位

- 作为项目级工作台，集中展示常用操作入口。
- 支持进入编辑模式，自定义部件布局。
- 部件变更会自动保存，适合长期维护个人工作面板。

## 当前可用部件

- `welcome`：欢迎类信息，适合作为首页提示。
- `quick-task`：快速触发常用任务，减少切换到任务页的频率。
- `news-feed`：聚合信息流，适合放团队公告或订阅源。
- `kill-port`：处理端口占用问题，适合开发服务频繁冲突的场景。

这些部件来自当前代码中的注册项，后续可以继续扩展。

## 常用工作流

1. 打开项目后先进入仪表盘确认当前上下文。
2. 点击“自定义”进入编辑模式。
3. 从右侧“添加部件”面板选择需要的部件。
4. 完成后退出编辑模式，系统会保存当前布局。

## 什么时候值得用仪表盘

- 你经常在同一项目反复执行同几类操作。
- 你希望把“快速任务”和“排障动作”收在一个入口里。
- 团队内部需要沉淀一套相对稳定的项目首页布局。
