# 任务执行

任务页用于集中执行项目脚本，例如 `dev`、`build`、`test`、`lint`。

## 典型工作流

1. 进入任务页，选择目标项目。
2. 从识别出的任务列表中选择脚本。
3. 执行后观察实时输出、退出码和耗时。

## 适合纳入任务面板的脚本

- 开发服务：`dev`、`start`
- 构建打包：`build`
- 质量校验：`test`、`lint`
- 定制自动化：团队内部的部署前检查、代码生成等

## 使用建议

- 优先保持脚本名称语义稳定，便于跨项目复用操作习惯。
- 把复杂命令收敛到 `package.json` scripts 中，不要依赖个人终端历史。
- 如果任务需要长时间运行，重点关注日志滚动和端口占用情况。

## 任务失败时先看什么

- 退出码是否为非零。
- 最近一段标准输出和错误输出。
- 是否存在端口冲突、环境变量缺失或 Node 版本不匹配。
