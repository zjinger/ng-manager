# 依赖管理

依赖页用于查看项目的运行依赖和开发依赖，并执行基础的安装、更新、卸载动作。

## 页面能力

- 按 `dependencies` 和 `devDependencies` 分组展示。
- 显示当前版本、声明版本和可获取的最新版本。
- 支持按名称搜索依赖。
- 可直接跳转到包主页或 npm 页面查看详情。

## 可以直接做什么

- 安装尚未安装的依赖。
- 更新已有依赖到最新版本。
- 卸载当前已安装的依赖。

这对排查“锁文件漂移”“团队本地环境不一致”很有帮助。

## 适合的使用场景

- 新接手项目时，快速看依赖结构是否完整。
- 构建失败后，确认缺失的是运行依赖还是开发依赖。
- 做一次小范围升级，先验证某个包是否可以单独更新。

## 使用建议

- 先在任务页停止正在运行的构建或开发服务，再执行依赖变更。
- 变更后建议重新跑一次 `build` 或 `test`，不要只看安装成功提示。
- 对核心基础库升级，优先在分支验证，不要直接在多人共享环境里批量升级。
