# 配置编辑

配置页面向“结构化配置维护”。它不是简单打开文本文件，而是尝试把配置文件映射成可编辑的表单模型。

## 页面能力

- 左侧显示配置域导航。
- 中间按 schema 渲染配置表单。
- 支持比较修改前后的差异。
- 支持保存写回。
- 支持直接在默认编辑器中打开对应配置文件。

## 当前实现特点

- 页面已经支持读取配置目录与配置域。
- 目前代码中对 `angular` 域有明确处理，其他域仍可以继续扩展。
- 保存后会重新加载模型，确保界面展示与实际落盘结果一致。

## 推荐用法

1. 先选择项目。
2. 打开配置页，选择需要操作的配置域。
3. 修改表单字段后先看“变更”。
4. 确认 diff 合理后再保存。
5. 如需深度修改，再用“打开配置文件”跳到外部编辑器。

## 什么时候用它

- 你要做低风险的结构化配置调整。
- 你希望减少手写 JSON / YAML 时的格式错误。
- 你需要先看“将会改哪些字段”，再决定是否保存。
