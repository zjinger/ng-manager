# ngm-hub 部署指南：SSH 免密登录配置

本文档用于指导团队成员在 **部署 ngm-hub 服务时配置 SSH Key 免密登录**。

配置完成后：

-   ssh 登录服务器无需输入密码
-   scp 上传文件无需输入密码
-   自动化部署脚本（deploy-server.js）可以无交互执行

适用于：

-   本地开发机：Windows / macOS / Linux
-   部署服务器：Linux
-   示例服务器：192.168.1.31

------------------------------------------------------------------------

# 1 为什么需要 SSH 免密登录

在部署流程中通常会执行多次命令：

    ssh
    scp

如果使用密码认证，每次连接都会要求输入密码，例如：

    ssh root@192.168.1.31
    scp file root@192.168.1.31:/opt/ngm-hub

部署脚本运行时会被迫暂停等待输入密码。

使用 **SSH 公钥认证（Key Login）** 可以彻底解决这个问题。

------------------------------------------------------------------------

# 2 SSH Key 工作原理

SSH Key 由两部分组成：

  类型   说明
  ------ ----------------
  私钥   保存在本地电脑
  公钥   保存在服务器

认证过程：

1.  客户端使用私钥
2.  服务器使用公钥验证
3.  验证成功即可登录

无需输入密码。

------------------------------------------------------------------------

# 3 检查本机是否已有 SSH Key

### Windows

    dir $env:USERPROFILE\.ssh

### macOS / Linux

    ls ~/.ssh

如果存在：

    id_ed25519
    id_ed25519.pub

说明已经存在 SSH Key。

如果没有，需要生成新的 Key。

------------------------------------------------------------------------

# 4 生成 SSH Key

推荐使用 **ed25519 算法**。

    ssh-keygen -t ed25519 -C "ngm-hub-deploy"

生成过程：

    Enter file in which to save the key

直接回车即可。

默认路径：

    ~/.ssh/id_ed25519

接下来：

    Enter passphrase

可以直接回车。

生成成功后会得到：

    ~/.ssh/id_ed25519
    ~/.ssh/id_ed25519.pub

Windows 路径示例：

    C:\Users\<username>\.ssh\

------------------------------------------------------------------------

# 5 查看公钥

需要将公钥复制到服务器。

### Windows

    type $env:USERPROFILE\.ssh\id_ed25519.pub

### macOS / Linux

    cat ~/.ssh/id_ed25519.pub

输出示例：

    ssh-ed25519 AAAAC3NzaC1lZDI1NTE5AAAAIxxxxxxxxxxxxxxxxxxx ngm-hub-deploy

复制整行。

------------------------------------------------------------------------

# 6 配置服务器 authorized_keys

登录服务器：

    ssh root@192.168.1.31

创建 SSH 目录：

    mkdir -p ~/.ssh
    chmod 700 ~/.ssh

编辑授权文件：

    vi ~/.ssh/authorized_keys

粘贴刚才复制的公钥。

保存后执行：

    chmod 600 ~/.ssh/authorized_keys

------------------------------------------------------------------------

# 7 测试免密登录

本地执行：

    ssh root@192.168.1.31 "echo ok"

如果输出：

    ok

并且 **没有要求输入密码**，说明配置成功。

------------------------------------------------------------------------

# 8 测试 scp 免密上传

    scp test.txt root@192.168.1.31:/tmp

如果没有输入密码提示，说明 scp 也已成功配置免密。

------------------------------------------------------------------------

# 9 指定 SSH Key（可选）

如果电脑存在多个 SSH Key，可以指定 Key：

    ssh -i ~/.ssh/id_ed25519 root@192.168.1.31

scp 示例：

    scp -i ~/.ssh/id_ed25519 file.tar.gz root@192.168.1.31:/opt/ngm-hub

------------------------------------------------------------------------

# 10 常见问题

## 1 仍然要求输入密码

检查权限：

    chmod 700 ~/.ssh
    chmod 600 ~/.ssh/authorized_keys

------------------------------------------------------------------------

## 2 authorized_keys 不生效

检查 sshd 配置：

    cat /etc/ssh/sshd_config

确保存在：

    PubkeyAuthentication yes
    AuthorizedKeysFile .ssh/authorized_keys

修改后重启 SSH：

    systemctl restart sshd

------------------------------------------------------------------------

## 3 Windows SSH Key 目录

默认位置：

    C:\Users\<username>\.ssh

------------------------------------------------------------------------

# 11 推荐团队使用方式

建议团队成员：

-   每个人使用自己的 SSH Key
-   公钥统一登记到服务器
-   不在脚本中存储服务器密码

部署脚本统一使用：

    ssh root@server
    scp file root@server:/path

即可自动完成部署。
